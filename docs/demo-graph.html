Conditioning a Gaussian graph

Pin a node, and a Schur complement ripples through the network.

Six beads on frictionless vertical rods, joined by springs along a chain — plus one shortcut edge from node 1 to node 4. This is a Gaussian graphical model: the precision matrix is the graph Laplacian of spring stiffnesses (a whisper of ground stiffness $\varepsilon$ keeps every node weakly anchored, so the system is well-posed even with nothing pinned). Click a node to pin it at its current value — that is conditioning: eliminating a variable from a Gaussian is exactly taking the Schur complement of its row and column from the precision matrix, and the remaining beads relax into the exact conditional mean.

Pin node 0, then node 5, and watch node 2 in the middle: it moves for both, not just its immediate neighbours — correlation propagates along every path, attenuated by distance and by the shortcut. Now pin node 1 and drag it: node 4 twitches immediately, skipping straight past nodes 2 and 3 — conditioning on a graph is not local the way the chain smoother was. Drag any unpinned node to nudge it and watch the network restore its shape.

pinned: none click a bead to pin/unpin; drag to move it

This is the multivariate generalisation the introduction promises in step 6: “pinning part of a spring network to the wall” is a Schur complement of the joint precision matrix, and the sister site schur.microprediction.org works out the block version of exactly this move for hierarchical portfolio allocation.
